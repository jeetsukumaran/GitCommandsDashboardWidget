Branching

git branch
List all local and remote branches.
git branch -r
List all local branches.
git branch new-branch-name
Create a new branch named new-branch-name, referencing the same point in history as the current branch.
git branch new-branch-name start-point-rev
Create a new branch named new-branch-name, referencing start-point-rev, which may be specified using a SHA1 commit ID, branch name, or tag.
git branch --track new-branch-name remote-name/remote-branch-name
Create a new tracking branch named new-branch-name, referencing, and pushing/pulling from, branch remote-branch-name, on remote repository remote-name.
git branch -d branch-name
Delete the local branch branch-name (fails if branch-name is not reachable from the current branch).
git branch -D branch-name
Force delete of the local branch branch-name (works even if branch-name is not reachable from the current branch).
git branch -d -r remote-name/remote-branch-name
Delete a "local remote" branch, i.e. a local tracking branch.
git push remote-name :heads/remote-branch-name
Delete a branch from a remote repository.
git checkout branch-name
Make the current branch branch-name, updating the working directory to reflect the version referenced by branch-name.
git checkout -b new-branch-name
Create a new branch new-branch-name, referencing the current point, and check it out (switch to it).
git checkout -b new-branch-name start-point-rev
Create a new branch new-branch-name, referencing start-point-rev, which may be specified using a SHA1 commit ID, branch name, or tag, and check it out (switch to it).
