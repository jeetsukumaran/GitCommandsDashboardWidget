Fix Mistakes / Undo

git reset
Resets the index (i.e., removes all changes staged for commit) but does not modify the working tree (i.e., the changes in the files are preserved).
git reset rev
Sets the current HEAD to the commit specified by rev, and resets the index but not the working tree.
git reset rev -- path
Restores file or files specified by path to version specified by rev, without moving HEAD.
git reset --soft rev
Sets the current HEAD to the commit specified by rev, but keep changes in the index for editing. For example, if something was forgotten or omitted in the previous commit, "git reset --soft HEAD^" will undo the last commit, and keep all changes in the index for editing and the next commit.
git reset --hard
Abandon all changes since last commit, restoring working tree to its last committed state (plus untracked files).
git reset --hard rev
As above, but with reference to commit specified by rev. For example, "git reset --hard ORIG_HEAD" will undo the most recent successful merge and any changes that occurred after. Useful for forgetting about the merge just done. If there are conflicts (the merge was not successful), use "git reset --hard" instead.
git commit --amend [-m msg]
Replace last commit message with msg if given, or open editor to edit last commit message.
