Fix Mistakes / Undo

All of the following can be modified by a "[-- path(s)]" argument clause, which limits the operations to the file or files found on path(s).
git reset
Resets the index but not the working tree (i.e., the changed files are preserved but not marked for commit) and reports what has not been updated.
git reset rev
Sets the current head to the commit specified by rev, which can be any SHA1 commit ID, branch name, or tag, and resets the index but not the working tree.
git reset --soft
Undo last commit, but keep all changes since in the index for editing.
git reset --soft rev
Sets the current head to the commit specified by rev, which can be any SHA1 commit ID, branch name, or tag, but keep changes in the index for editing. For example, if something was forgotten or omitted in the previous commit, "git reset --soft HEAD^" will undo the last commit, and keep all changes in the index for editing and the next commit.
git reset --hard
Abandon all changes since last commit, restoring working tree to its last committed state (plus untracked files).
git reset --hard rev
As above, but with reference to commit specified by rev, which can be any SHA1 commit ID, branch name, or tag. For example, "git reset --hard ORIG_HEAD" will undo the most recent successful merge and any changes that occurred after. Useful for forgetting about the merge just done. If there are conflicts (the merge was not successful), use "git reset --hard" instead.
git commit --amend [-m msg]
Replace last commit message with msg if given, or open editor to edit last commit message.
